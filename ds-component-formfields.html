Form Fields

Form fields allow the user to input data into our UI. A typical use case is in forms or during order entry. It should be clear to the user that a form field can be used to input information and each one should be clearly labeled to differentiated one from another. Allied’s form fields also have different states depending on the user’s content input state.

Form Field Types

The most common form field types are:

Outlined Form Fields

Our form field for standard use. This form field is designed to have a clear affordance for interaction and also keep the label hinting through-out the input process. It is also equipped with multiple states for the user.

Outline Form Fields with Icons

The same look as our normal outlined form field except it adds a visual element of an icon to help with scannability for more complex and dense form layouts.

Dropdown Fields

Used for the selection of preset nested options.

Text Field Area

Used for taller text areas that need to wrap and overflow. They are a fixed height and create a vertical scroll once the content reaches the bottom of the field.

Form Field Anatomy

1. Icon - Optional Use
2. Border – 1px stroke - 2px corner
3. Hint Label – 12px - Color: #979797
4. Helper Text - 9px - Color: #979797 or #9E0000
5. Focus Label - 9px - Color: #008baa
6. Cursor – Indicates the current location of the text input.
